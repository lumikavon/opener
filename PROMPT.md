开发一个可离线运行的桌面启动器应用“Opener”，用 Tauri(v2) + 原生 HTML5/CSS/JS + TailwindCSS(本地构建，不使用在线 CDN) + SQLite3（本地数据库）实现。你的输出应包含：可直接编译运行的完整源代码、必要的资源文件、数据库初始化脚本/迁移方案、以及一份可操作的安装/打包/使用说明文档（README）。不要只写示例片段，要求项目结构完整、功能可用、可验证。

一、目标平台与基本约束
- 平台：Windows 10/11 必须支持；macOS、Linux 以同一套代码可打包为目标（至少提供打包脚本/命令与配置）。
- 离线要求：前端所有依赖（Tailwind 构建产物、图标库等）必须随应用打包；禁止引用任何外部 CDN URL；应用在无网络环境可正常启动、搜索、配置、导入导出。
- 安全：执行“命令/脚本/SSH”等高危操作前必须有明确的用户确认开关与提示；敏感配置（如 SSH 密钥/密码）不得明文存储，给出使用系统凭据或加密存储的实现（按平台能力选择：优先 OS Keychain/Credential Vault；否则使用本地加密并说明密钥来源）。
- 交互性能：搜索输入后 150ms 内响应（本地数据规模 5,000 条配置项下），列表滚动与键盘导航不卡顿。
- 代码质量：Rust 端与前端均提供清晰的模块划分、注释仅用于解释关键设计；提供最少一组可运行的自动化验证（单元测试或集成测试）用于校验核心逻辑（如搜索、导入导出、数据库读写）。

二、UI/交互（无系统标题栏）
1) 主窗口（无系统标题栏）
- 窗口：使用 Tauri 无边框窗口；自定义顶部可拖拽区域（drag region），并提供最小化/关闭按钮（位于右上角）。
- 顶部一行：输入框（placeholder：“Search…”）+ 输入框右侧“设置”按钮（齿轮图标）+ “搜索/回车”触发按钮（可选，回车必须可触发）。
- 输入框行为：
  - 支持实时模糊搜索：输入变化即触发（带防抖 150ms）。
  - 支持键盘：↑/↓ 选择结果，Enter 执行选中项，Esc 清空输入或关闭弹窗（定义清晰规则并实现）。
- 结果列表：
  - 在输入框下方动态展示；每条显示：图标（可选）、名称、类型标签（App/URL/File/Dir/Cmd/WSL/SSH/Script/Shortcut/AHK）、简短描述或路径。
  - 选中项高亮；鼠标点击可执行；右键/更多按钮可打开“编辑/禁用/删除/复制路径”等操作菜单（至少实现编辑与删除）。
  - 空结果时显示“无匹配结果”占位。

2) 设置模态窗口（点击齿轮打开）
- 形式：模态对话框，支持 Esc 关闭；支持多标签页/侧边栏导航，至少包含以下页：
  A. “条目管理”（核心）
  B. “快捷键”
  C. “显示与排序”
  D. “脚本模板”
  E. “导入/导出”
  F. “关于/版本”
- 模态内所有表单项需要校验与错误提示（如必填、路径不存在、URL 非法等）。

三、数据模型与可配置项（SQLite）
1) SQLite 表设计（必须给出 schema）
- entries：启动条目表，字段至少包含：
  - id（主键）、name、type（枚举：app/url/file/dir/cmd/wsl/ssh/script/shortcut/ahk）
  - target（路径/URL/命令主串/脚本内容引用）
  - args（可选）、workdir（可选）、icon_path（可选）、tags（可选）、enabled（bool）
  - created_at、updated_at
- hotkeys：快捷键绑定表，字段至少包含：id、entry_id、accelerator（如 Ctrl+Alt+O）、scope（global/app）、enabled
- settings：全局设置表，存储显示方式、排序策略、默认执行行为、危险操作确认开关等。
- script_templates：脚本模板表，字段至少包含：id、name、language/type（shell/powershell/python等）、template_content、variables_schema（用于渲染表单）

2) 数据访问
- Rust 端封装 DB 访问层（CRUD + 查询）；提供迁移机制（首次启动自动创建表；版本升级可迁移）。
- 搜索查询要求：
  - 模糊匹配 name/target/tags（至少 name + target）。
  - 排序可配置：相关度优先/名称/最近使用（若实现最近使用需记录 last_used_at）。
  - 结果限制：默认最多 50 条，支持配置。

四、功能需求（必须逐项落地）
1) 搜索与执行
- 用户输入关键字 -> 返回匹配 entries 列表 -> 点击/回车执行：
  - URL：用系统默认浏览器打开
  - File：用系统默认程序打开
  - Dir：在文件管理器中打开
  - App：启动可执行文件（可带参数、工作目录）
  - Cmd：执行命令（可选择是否显示终端窗口；Windows 下说明实现方式）
  - WSL：配置发行版（可选）、命令；通过 wsl.exe 执行
  - SSH：配置 host/user/port/identityKey；执行 ssh 连接（可选在新终端打开）
  - Script：从模板生成或自定义脚本；执行前可渲染变量
  - Shortcut：解析 .lnk（Windows）或对应平台快捷方式（若跨平台难度大，可限定 Windows 并在文档声明）
  - AHK：配置 .ahk 脚本路径或内容；执行（仅 Windows，需检测 AutoHotkey 是否安装或允许用户指定 AHK 可执行文件路径）
- 执行前的安全确认：
  - 对 Cmd/WSL/SSH/Script/AHK 默认弹确认（可在设置中关闭或对单条关闭）。
  - 确认弹窗显示：将要执行的完整命令行/目标与来源条目。

2) 条目配置（在“条目管理”页）
- 支持新增/编辑/删除/启用禁用 entry。
- 新增时选择类型，不同类型显示不同字段：
  - URL：URL 输入与校验
  - File/Dir/App：选择器（打开系统文件/目录选择对话框）+ 手动输入
  - Cmd/WSL/SSH/Script/AHK：多行输入、参数字段、工作目录、环境变量（可选）
- 图标：
  - 允许为条目指定 icon_path（本地文件），并在列表显示。
  - 若为 App/File，可尝试自动提取图标（实现能力允许则做；否则提供手动指定并在文档说明限制）。

3) 快捷键绑定（“快捷键”页）
- 支持为条目绑定快捷键（至少应用内快捷键；若实现全局快捷键需使用 Tauri 对应能力并声明权限）。
- 支持冲突检测：同一 scope 下不允许重复；提供冲突提示与替换选项。
- 支持一个条目多个快捷键或限制为一个（给出明确规则并实现）。

4) 搜索结果显示方式（“显示与排序”页）
- 可配置项至少包含：
  - 图标大小：16/24/32/48
  - 每条显示字段开关：显示路径/显示类型标签/显示描述
  - 排序策略：相关度/名称/最近使用
  - 最大展示条数：10/20/50/100
- 配置实时生效并写入 settings 表。

5) 预定义脚本模板（“脚本模板”页）
- 内置至少 5 个模板（以本地 JSON/SQLite 初始化数据提供），例如：
  - 打开项目：输入路径变量并执行（shell/powershell）
  - Git 拉取：repo 路径 + 分支
  - 启动开发服务器：端口 + 工作目录
  - SSH 连接：host/user/port（生成 ssh 命令）
  - WSL 执行：distro + command
- 模板支持变量：
  - 定义变量名、类型（string/number/path/choice）、默认值、必填、校验规则
  - UI 自动生成表单；渲染成最终脚本/命令预览
- 用户可基于模板“一键创建条目”，并保存为 Script/Cmd/WSL/SSH 类型之一。

6) 配置导入/导出（“导入/导出”页）
- 导出：将 entries + hotkeys + settings + templates（可选）导出为单一文件（建议 JSON），并允许用户选择保存路径。
- 导入：选择文件 -> 校验结构与版本 -> 提供合并策略：
  - 覆盖全部 / 仅新增 / 按名称去重合并
- 导入导出文件需包含 schema_version，便于未来升级迁移。
- 导入失败需提供明确错误信息（缺字段、版本不兼容、JSON 解析失败等）。

7) 打包与图标
- 提供可执行的打包流程：
  - Windows：生成 .msi（或至少 nsis + msi 二选一，优先 msi），给出命令
  - macOS：生成 .dmg 或 .app 打包说明
  - Linux：提供 .deb/.AppImage 其一及说明
- 图标：
  - 提供应用图标源文件（如 SVG/PNG）与通过 Tauri 工具生成各平台图标的脚本/命令。
  - 确保 `tauri.conf` 等配置正确引用。
